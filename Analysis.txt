Analysis: The significant differece in latencies between the two instances can be attributed to the distance the eruopean server is when compared to the US server. In terms of the differnece observed between the register and list endpoints, it can be noticed that register takes longer than list for both instances. This difference can be attributed to register needing to send more information in its HTTP Post request and the database having to insert a new entity. Both of these take more time than list which is a HTTP GET request and simple database query. If Firestore is replicating the database then that is something that will also cause the /register endpoint to take longer as it will need to wait for a set number of replicas to respond before responding back to the user while list can just return the data immediately as it is eventually consistent. The consistency experiment showed that GCP has very little replication lag as all the usernames were found when list was run in part B for the european vm.
